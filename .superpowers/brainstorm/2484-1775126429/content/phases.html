Implementation Phases

I've grouped the 7 features into 3 phases by dependency. Does this ordering feel right?

Phase 1: Rich Message Rendering

Foundation — parse agent responses into structured components instead of raw text

What changes
Feature F — Rich Flight/Hotel Cards

Parse tool_result events. When flights/hotels/experiences come back, render structured cards with airline logos, star ratings, prices, and photos instead of letting Claude describe them in text.

Feature C — Quick Reply Chips

Parse agent text for yes/no and multiple-choice questions. Render clickable chips below the message. Clicking sends the response automatically.

Feature A — Clickable Option Cards

When agent presents numbered options (3 flights, 2 hotels), render as selectable cards. User clicks to select, submit button to confirm. No typing.

Backend: Add airline_logo to FlightResult, photos to ExperienceResult, images to HotelResult

Phase 2: Visual Media

Depends on Phase 1 cards — adds images and maps to the card components

What changes
Feature B — Google Places Photos

Add places.photos to Google Places field mask. Proxy photo URLs through the backend (API key protection). Render in experience/restaurant cards.

Feature E — Map Preview Cards

When a location has lat/lng (from Places API), render a static Google Maps image. Small card with a pin showing where the hotel/restaurant is.

Backend: Add photo proxy endpoint, include lat/lng in ExperienceResult, add Google Maps Static API key

Phase 3: Inline Widgets

Polish layer — widgets that appear within the chat flow

What changes
Feature D — Inline Budget Tracker

After each tool_result that adds a cost, inject a mini budget bar into the chat. Shows allocated vs. remaining as a visual progress bar with labels.

Feature G — Day-by-Day Timeline

When the agent builds a full itinerary, parse the day-by-day structure and render as a visual timeline with collapsible day cards and time slots.

Frontend only — no backend changes needed
